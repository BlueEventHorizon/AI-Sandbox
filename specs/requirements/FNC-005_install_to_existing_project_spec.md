# FNC-005 既存プロジェクトへのインストール 要件定義書

## 概要

AI DevContainer 環境を既存のプロジェクトに追加するためのインストール機能を提供する。自動インストールスクリプトにより、必要なファイルのコピーを行う。AI コーディングアシスタントはコンテナイメージにプリインストールされるため、プロジェクト側の依存関係管理（package.json 等）は不要。

## 前提条件

- インストール先のディレクトリが存在すること
- AI DevContainer のリポジトリがクローンされていること

## 要件一覧

### インストールステップ要件

インストールスクリプトは以下のステップを順番に実行する:

1. **`.devcontainer/` のコピー**: DevContainer 設定ディレクトリ全体をコピーする
2. **`start-container.sh` のコピー**: コンソール起動スクリプトをコピーし、実行権限を付与する

※ Makefile（macOS Docker セットアップ）は既存プロジェクトにコピーしない。macOS で Docker 環境が未構築の場合は、AI DevContainer リポジトリで `make install` を実行する。

### バックアップ要件

- `--backup` オプション指定時、既存ファイルを上書き前にバックアップする
- バックアップファイル名にはタイムスタンプを付与する

### 上書き確認要件

- 既存ファイルがある場合、`--backup` 未指定時は上書きの確認プロンプトを表示する
- 必須ファイル（`.devcontainer/`）の上書きをユーザーが拒否した場合はインストールを中止する
- オプションファイル（`start-container.sh`）の上書きをユーザーが拒否した場合は該当ファイルをスキップし、インストールを継続する

### エラーケース

| 条件 | 動作 |
| --- | --- |
| ターゲットディレクトリ未指定 | エラーを表示して終了する |
| ターゲットディレクトリが存在しない | エラーを表示して終了する |
| 必須ファイルの上書き確認で拒否 | インストールを中止する |
| start-container.sh の上書き確認で拒否 | スキップし、インストールを継続する |

# FNC-005 既存プロジェクトへのインストール 要件定義書

## 概要

AI DevContainer 環境を既存の Node.js プロジェクトに追加するためのインストール機能を提供する。自動インストールスクリプトにより、必要なファイルのコピーと設定のマージを行う。

## 前提条件

- インストール先のディレクトリが存在すること
- AI DevContainer のリポジトリがクローンされていること

## 要件一覧

### インストールステップ要件

インストールスクリプトは以下の5ステップを順番に実行する:

1. **`.devcontainer/` のコピー**: DevContainer 設定ディレクトリ全体をコピーする
2. **`package.json` の処理**:
   - 既存ファイルがない場合: そのままコピー
   - 既存ファイルがある場合: devDependencies と scripts を既存ファイルにマージする
3. **`.gitignore` の更新**: `node_modules/` が含まれていなければ追加する
4. **`start-container.sh` のコピー**: コンソール起動スクリプトをコピーし、実行権限を付与する
5. **`Makefile` のコピー（オプション）**: macOS ユーザー向け、ユーザーの確認後にコピーする

### バックアップ要件

- `--backup` オプション指定時、既存ファイルを上書き前にバックアップする
- バックアップファイル名にはタイムスタンプを付与する（例: `package.json.backup.20260321_231500`）

### 上書き確認要件

- 既存ファイルがある場合、`--backup` 未指定時は上書きの確認プロンプトを表示する
- 必須ファイル（`.devcontainer/`）の上書きをユーザーが拒否した場合はインストールを中止する
- オプションファイル（`Makefile`, `start-container.sh`）の上書きをユーザーが拒否した場合は該当ファイルをスキップし、インストールを継続する

### package.json マージ要件

- 自動マージツールが利用可能な場合: 自動マージを実行する
  - 既存の devDependencies に AI コーディングアシスタントの依存関係を追加
  - 既存の scripts に AI コーディングアシスタントの実行スクリプトを追加
  - 既存の設定はそのまま保持する
- 自動マージツールが利用できない場合: 手動追加の手順を表示する

### エラーケース

| 条件 | 動作 |
| --- | --- |
| ターゲットディレクトリ未指定 | エラーを表示して終了する |
| ターゲットディレクトリが存在しない | エラーを表示して終了する |
| 必須ファイルの上書き確認で拒否 | インストールを中止する |
| オプションファイルの上書き確認で拒否 | 該当ファイルをスキップし、インストールを継続する |
| 自動マージに失敗 | 手動追加の手順を表示し、処理を継続する |
| 自動マージツールが未インストール | 手動追加の手順を表示し、処理を継続する |

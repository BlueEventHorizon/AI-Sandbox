# FNC-002 コンテナ起動（ターミナル） 要件定義書

## 概要

`setup-sandbox.sh` でターゲットの git プロジェクトのパスを指定し、AI コーディングアシスタントがプリインストールされたコンテナを起動してシェルに入る。ターゲットプロジェクトへのファイルコピーは不要。

**優先度**: 1（最優先）

## 前提条件

- Docker（または互換ランタイム）が起動していること
- AI Sandbox リポジトリがクローン済みであること
- ターゲットプロジェクトが git で管理されていること

## 要件一覧

### 起動要件

- `setup-sandbox.sh <ターゲットプロジェクトのパス>` でコンテナを起動する
- AI Sandbox リポジトリのディレクトリから実行する
- ターゲットプロジェクトへのファイルコピーは行わない

### イメージビルド要件

- イメージが存在しない場合は自動的にビルドする
- イメージが存在する場合はそのまま起動する
- `--rebuild` オプションでイメージを強制的に再ビルドして起動する

### コンテナ起動後の動作要件

- コンテナ起動後、同じターミナルがそのままコンテナ内のシェルに切り替わる
- プロンプトが `/workspace $` に変わり、ターゲットプロジェクトの全ファイルが見える
- `claude` および `codex` コマンドが即時利用可能であること
- `exit` コマンドでコンテナから抜けることができる
- コンテナ終了時に自動的にコンテナを削除する（一時利用）

### ボリュームマウント要件

- ターゲットプロジェクトのルートディレクトリをコンテナ内の `/workspace` にマウントする
- ホスト側のファイル変更がコンテナ内にリアルタイムで反映される
- コンテナ内での変更がホスト側にリアルタイムで反映される（AI が行った変更が git プロジェクトに残る）

### 複数プロジェクトの同時実行要件

- 異なるターミナルから異なるターゲットプロジェクトに対して同時にコンテナを起動できること
- 各コンテナは独立したインスタンスとして動作し、互いに干渉しない
- 全コンテナが同一のイメージを共有する（イメージのビルドは1回のみ）

### ネットワーク要件

- コンテナからインターネットへのアウトバウンド通信が可能であること（AI ツールの API 通信に必要）

### データ永続化要件

- Claude Code の認証情報・設定（`~/.claude/`）を Docker Named Volume で永続化する
- コンテナ終了・再起動を跨いで認証情報が保持されること（毎回のログインが不要）
- `colima stop` → `colima start` を跨いでも Volume が保持されること
- ターゲットプロジェクトへの変更は bind mount 経由でホスト側に即時反映される（Volume 不要）

### ヘルプ要件

- `-h` または `--help` オプションで使い方を表示する

### プラットフォーム制約

- `setup-sandbox.sh` はシェルスクリプト（sh）であり、macOS および Linux で動作する

### エラーケース

| 条件 | 動作 |
| --- | --- |
| Docker が未インストール | エラーを表示して終了する |
| Docker が起動していない | エラーを表示して終了する |
| ターゲットパスが指定されていない | 使い方を表示して終了する |
| ターゲットパスが存在しない | エラーを表示して終了する |
| ターゲットパスが git リポジトリでない | エラーを表示して終了する |
| Dockerfile が見つからない（AI Sandbox リポジトリ内） | エラーを表示して終了する |
| イメージのビルドに失敗 | エラーを表示して終了する |
| 不明なオプション | 使い方を表示して終了する |

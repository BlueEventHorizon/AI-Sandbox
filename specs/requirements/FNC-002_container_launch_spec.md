# FNC-002 コンテナ起動（コンソール） 要件定義書

## 概要

VS Code を使わずに、コマンドラインから起動スクリプトを実行してコンテナを起動し、シェルに入る。コンテナ起動時に依存関係を自動インストールし、AI コーディングアシスタントのコマンドが実行可能な状態にする。

## 前提条件

- Docker（または互換ランタイム）が起動していること
- プロジェクトルートディレクトリで実行すること

## 要件一覧

### 起動モード要件

- **通常モード**: DevContainer イメージを使用してコンテナを起動する
  - イメージが存在しない場合は自動的にビルドする
  - イメージが存在する場合はそのまま起動する
- **シンプルモード（--node）**: ビルド不要で、公式 Node.js イメージを使って起動する
- **再ビルドモード（--rebuild）**: イメージを強制的に再ビルドして起動する

### 操作要件

- 起動スクリプト実行後、コンテナ内のシェルに自動的に入る
- コンテナ起動時に npm 依存関係を自動インストールする
- `exit` コマンドでコンテナから抜けることができる
- コンテナ終了時に自動的にコンテナを削除する（一時利用）
- ヘルプオプション（-h, --help）で使い方を表示する

### ボリュームマウント要件

- カレントディレクトリをコンテナ内の `/workspace` にマウントする
- ホスト側のファイル変更がコンテナ内にリアルタイムで反映される

### エラーケース

| 条件 | 動作 |
| --- | --- |
| Docker が未インストール | エラーを表示して終了する |
| Docker が起動していない | エラーを表示して終了する |
| Dockerfile が見つからない | エラーを表示して終了する |
| イメージのビルドに失敗 | エラーを表示して終了する |
| 不明なオプション | 使い方を表示して終了する |

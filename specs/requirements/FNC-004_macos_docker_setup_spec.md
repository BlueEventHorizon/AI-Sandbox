# FNC-004 macOS Docker 環境セットアップ 要件定義書

## 概要

macOS ユーザー向けに、Docker Desktop の代替として Colima（OSS、無料）を使った Docker 環境の自動セットアップを提供する。Homebrew を前提に、必要なコンポーネントを自動でインストール・起動する。

## 前提条件

- macOS であること
- Homebrew がインストールされていること

## 要件一覧

### インストール要件

`make install` コマンドで以下のコンポーネントを自動セットアップする:

| コンポーネント | 役割 | インストール方法 |
| --- | --- | --- |
| Docker CLI | Docker コマンド実行 | Homebrew |
| Colima | Docker ランタイム（Docker Desktop 代替） | Homebrew |
| Docker Buildx | マルチアーキテクチャビルド対応 | 公式リリースからインストール |

- 各コンポーネントが既にインストール済みの場合はスキップする
- インストール後、Colima を自動起動する
- Colima が既に起動済みの場合はスキップする

### アーキテクチャ対応要件

- Intel Mac（x86_64）と Apple Silicon（arm64）の両方に対応する
- Buildx プラグインのダウンロード時にアーキテクチャを自動検出する

### アンインストール要件

`make uninstall` コマンドで以下を実行する:

- Colima の停止・削除・アンインストール
- Colima のデータディレクトリの削除
- Docker Buildx プラグインの削除
- Docker CLI のアンインストール

### 日常操作要件

- `colima start`: Colima を起動する（2回目以降の利用時）
- `colima stop`: Colima を停止する

### プラットフォーム制約

- macOS 以外のプラットフォームで `make install` を実行した場合、エラーメッセージを表示して終了する

### エラーケース

| 条件 | 動作 |
| --- | --- |
| macOS 以外で実行 | エラーを表示して終了する |
| Homebrew が未インストール | エラーを表示して終了する |
| Docker のインストール失敗 | エラーを表示して終了する |
| Colima のインストール失敗 | エラーを表示して終了する |
| Buildx のインストール失敗 | エラーを表示して終了する |
| Colima の起動失敗 | エラーを表示して終了する |
| 未対応アーキテクチャ | エラーを表示して終了する |

# FNC-001 DevContainer 環境構築 要件定義書

## 概要

VS Code の DevContainers 拡張機能と連携し、Alpine Linux ベースの Node.js コンテナ環境を自動構築する。コンテナ起動時に依存関係をインストールし、AI コーディングアシスタントのコマンドが実行可能な状態にする。

## 前提条件

- Docker（または互換ランタイム）が起動していること
- VS Code と DevContainers 拡張機能がインストールされていること

## 要件一覧

### コンテナ環境要件

- ベースイメージとして Alpine Linux ベースの Node.js 24 環境を使用する
- コンテナ内の実行ユーザーは非 root ユーザー（node）とする
- プロジェクトルートがコンテナ内のワークスペースとしてマウントされ、一貫したパスで参照可能であること

### 依存関係管理要件

- コンテナ起動時（ビルド時ではなく）に npm 依存関係を自動インストールする
- Dockerfile の変更なしで依存関係の更新が可能であること

### VS Code 連携要件

- VS Code でプロジェクトを開いた際に、DevContainer での再オープンを提案する
- コンテナ接続後、統合ターミナルからコンテナ内のシェルを利用できる
- AI コーディングアシスタント用の VS Code 拡張機能を自動推奨する
  - Google Gemini Code Assist
  - Anthropic Claude Code

### シェル環境要件

- AI コーディングアシスタント（gemini, claude, codex）のコマンドエイリアスを提供する
- `node_modules/.bin` に PATH を通し、ローカルインストールされたツールを直接実行可能にする

### エラーケース

| 条件 | 動作 |
| --- | --- |
| Docker が起動していない | DevContainer の起動に失敗する（VS Code がエラーを表示） |
| DevContainers 拡張機能が未インストール | DevContainer の認識・起動ができない |

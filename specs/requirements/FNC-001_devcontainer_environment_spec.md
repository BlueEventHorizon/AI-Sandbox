# FNC-001 DevContainer 環境構築 要件定義書

## 概要

VS Code の DevContainers 拡張機能と連携し、Alpine Linux ベースのコンテナ環境を自動構築する。AI コーディングアシスタントはコンテナイメージにプリインストールされており、起動後すぐに利用可能な状態となる。

## 前提条件

- Docker（または互換ランタイム）が起動していること
- VS Code と DevContainers 拡張機能がインストールされていること

## 要件一覧

### コンテナ環境要件

- ベースイメージとして Alpine Linux を使用する
- コンテナ内の実行ユーザーは非 root ユーザーとする
- プロジェクトルートがコンテナ内のワークスペースとしてマウントされ、一貫したパスで参照可能であること

### AI コーディングアシスタントのプリインストール要件

- AI コーディングアシスタントはコンテナイメージのビルド時にインストールする
- コンテナ起動時に追加のインストール処理（npm install 等）を必要としない
- ツールの更新はコンテナイメージの再ビルドで行う

### VS Code 連携要件

- VS Code でプロジェクトを開いた際に、DevContainer での再オープンを提案する
- コンテナ接続後、統合ターミナルからコンテナ内のシェルを利用できる
- AI コーディングアシスタント用の VS Code 拡張機能を自動推奨する
  - Anthropic Claude Code

### 対応プラットフォーム要件

- 以下の OS 環境でコンテナが正常に起動し、AI コーディングアシスタントが動作すること:
  - macOS（Intel / Apple Silicon）
  - Linux（x86_64）
- 各 OS で同一の Dockerfile からビルドしたイメージが同一の動作をすること

### シェル環境要件

- AI コーディングアシスタント（claude, codex）のコマンドに PATH が通っていること
  - 検証方法: `which claude && which codex` が各コマンドのパスを出力し、終了コード 0 で完了すること
- コンテナ起動後、追加設定なしでコマンドを実行できること
  - 検証方法: `claude --help` および `codex --help` が終了コード 0 で完了すること

### エラーケース

| 条件 | 動作 |
| --- | --- |
| Docker が起動していない | DevContainer の起動に失敗する（VS Code がエラーを表示） |
| DevContainers 拡張機能が未インストール | DevContainer の認識・起動ができない |
